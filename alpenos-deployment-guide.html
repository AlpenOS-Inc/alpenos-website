Onboarding Form
Deployment Guide

Step-by-step instructions for deploying the Design Partner onboarding wizard to your alpenos.io site. Covers file setup, Netlify form capture, Make.com automation, and Google Sheets staging. No coding experience required.

01
// Phase 1
File Setup
~10 min
Step 1.1
Download the form file
Required

Claude generated a file called alpenos-onboarding.html. Download it from the chat and save it somewhere you can find it — your Desktop is fine for now.

// Note This is a single self-contained HTML file. It has no dependencies — no separate CSS or JS files needed.
Step 1.2
Add the file to your GitHub repo
Required

Since your Netlify site is connected to a GitHub repo, you need to put the file there. Here's exactly how:

  1. Go to github.com and open the repo connected to alpenos.io
  2. In your repo file list, click Add file → Upload files
  3. Drag alpenos-onboarding.html into the upload box
  4. Make sure the file is in the root folder of the repo (same level as your main index.html)
  5. Scroll down, type a commit message like "Add onboarding form", and click Commit changes
// Result Once committed, Netlify will automatically detect the push and redeploy your site — usually takes under 60 seconds. Your form will be live at alpenos.io/alpenos-onboarding
Step 1.3
Set your partner access codes
Config

Before you go live, change the default access codes to ones you control. Open alpenos-onboarding.html in any text editor (Notepad on Windows, TextEdit on Mac, or VS Code if you have it).

Find this line near the top of the <script> section:

// Find this line (~line 520)
const VALID_CODES = ['ALPEN-2026', 'SUMMIT-01', 'PILOT-001'];

Replace those strings with your own codes. Recommended format: one code per design partner so you can track who's onboarding.

// Example — one code per pilot partner
const VALID_CODES = [
  'SUMMIT-OE-01',   // Summit Outdoor Education
  'RIDGE-GUIDE-01', // Ridge Line Guiding
  'CRESTLINE-01',   // Crestline Youth Programs
];
// Important Don't use codes that are easy to guess. Anyone with the URL and a valid code can submit. Keep codes to trusted partners only and rotate them if needed.
02
// Phase 2
Netlify Configuration
~15 min
Step 2.1
Push to GitHub and watch Netlify deploy
Required

After committing the file in Step 1.2, go to your Netlify dashboard at app.netlify.com.

  1. Click on your alpenos.io site
  2. You'll see a Deploy log — watch for a green "Published" status
  3. Once published, open a new browser tab and go to: https://alpenos.io/alpenos-onboarding
  4. You should see the access code gate screen
// If the page shows a 404 error Netlify may need the file extension in the URL. Try alpenos.io/alpenos-onboarding.html — or add a redirect rule (ask Claude to help with that if needed).
Step 2.2
Verify Netlify detected the form
Required

Netlify automatically scans your HTML files for forms with the netlify attribute. You need to confirm it detected yours.

  1. In your Netlify dashboard, go to your site
  2. Click the Forms tab in the top navigation
  3. You should see a form named "alpenos-onboarding" listed
// If the form does NOT appear Go to Deploys, find your latest deploy, and click Clear cache and redeploy. Netlify scans for forms at deploy time — a cache-busted redeploy usually fixes it.
// Free tier limit Netlify Forms free tier allows 100 form submissions/month. For 6 design partners this is more than sufficient for V0. You will not need to upgrade.
Step 2.3
Set up Netlify email notifications
Set Once

Configure Netlify to email you every time a partner submits the form. This is your immediate alert before Make.com processes the data.

  1. In Netlify dashboard → Forms
  2. Click on alpenos-onboarding
  3. Click Form notifications or go to Site settings → Forms → Form notifications
  4. Click Add notification → Email notification
  5. Enter your email address and select the alpenos-onboarding form
  6. Save
03
// Phase 3
Make.com Automation
~30 min
Step 3.1
Create your Make.com account
Set Once
  1. Go to make.com and click Get started free
  2. Sign up with your AlpenOS admin email
  3. On the free plan you get 1,000 operations/month — more than enough for V0 pilot onboarding
  4. Once logged in, you'll see the Make dashboard. Click Create a new scenario
// What is Make.com? Make is a no-code automation tool. Think of it as the routing layer that listens for a Netlify form submission and then writes a new row into your Google Sheets staging table. No code required.
Step 3.2
Build the Netlify → Google Sheets scenario
Required

This is the core automation. Follow these steps exactly inside Make.com:

  1. In the new scenario canvas, click the large "+" circle to add your first module
  2. Search for "Netlify" and select it
  3. Choose the trigger: "Watch Form Submissions"
  4. Click Add to connect your Netlify account — it will ask you to log in to Netlify and authorize Make
  5. Once connected, select your site (alpenos.io) and form (alpenos-onboarding)
  6. Click OK

Now add the second module (the output):

  1. Click the "+" after the Netlify module to add the next step
  2. Search for "Google Sheets" and select it
  3. Choose the action: "Add a Row"
  4. Connect your Google account when prompted
  5. Select your Staging Spreadsheet (you'll create this in Phase 4 — come back to finish this step after Phase 4)
  6. Select the correct Sheet tab (e.g. "PARTNER_STAGING")
// Tip Leave this scenario tab open while you do Phase 4. You'll come back to map the fields in Step 3.3.
Step 3.3
Map form fields to sheet columns
Required

After connecting Google Sheets in Step 3.2, Make will show you a list of columns from your sheet. You need to map each one to the corresponding field from the Netlify form submission.

// Phase 4
Google Sheets Staging
~15 min
Step 4.1
Create the Staging Sheet
Set Once

This is the Airlock — raw partner data lands here first before you manually promote it into your master AlpenOS workbook.

  1. Go to sheets.google.com and create a new spreadsheet
  2. Name it: AlpenOS — Partner Staging
  3. Rename the default tab to: PARTNER_STAGING
  4. Add the column headers in Row 1 exactly as listed in Step 4.2 below
  5. Add two extra columns at the end: PartnerID_Assigned and Status — these you fill in manually after review
Step 4.2
Column header map
Config

Copy this header row exactly into Row 1 of your PARTNER_STAGING tab. Column names must match the Make.com field mapping from Step 3.3.

05
// Phase 5
Launch & Test
~20 min
Step 5.1
End-to-end test before sending to partners
Required — Do this first

Do a full test submission yourself before handing the link to any design partner. This catches broken field mapping before it matters.

  1. Go to alpenos.io/alpenos-onboarding
  2. Enter one of your access codes
  3. Fill out all 5 steps with fake but realistic data
  4. Submit
  5. Check: Netlify Forms — does the submission appear under the alpenos-onboarding form?
  6. Check: Make.com — did the scenario trigger? (Check the scenario run history)
  7. Check: Google Sheets PARTNER_STAGING — did a new row appear with all fields populated?
  8. Check: Your email — did you receive the Netlify notification?
// Common issue If Make.com didn't trigger, the scenario may be set to run on a schedule instead of instantly. In Make.com, open your scenario, click Scheduling, and set it to "Immediately as data arrives".
Step 5.2
Send the partner link
Required

Once the test passes, you're ready. Send each design partner their unique access code with the link. Recommended message format:

// Partner outreach template
Subject: Your AlpenOS Onboarding Portal is Ready

Hey [Name],

Your AlpenOS design partner onboarding portal is live.

Link: https://alpenos.io/alpenos-onboarding
Access Code: [THEIR-CODE]

The form takes about 10–15 minutes. It covers your org
profile, operating areas, op types, and risk configuration
— everything we need to build your V0 environment.

Once you submit, I'll follow up within 24 hours to schedule
your config call and send individual invite links to your
team members.

—[Your name]
// White glove touch Offer to do a screen-share walkthrough with your first 1–2 partners. The form is intuitive but your first design partners are also your first feedback loop. Their friction points on the form = friction points in the product.
